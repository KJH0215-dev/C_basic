## 오늘 공부한 내용 (2026-3-9)

### 프로그래밍의 개념
-컴퓨터는 0, 1만을 읽을 수 있음 그래서 인간의 언어(Natural language)로 나타내기 위해서는 번역과정이 필수적

-컴퓨터 언어 
        -저급 언어(기계어에 가까운 언어): <ex>어셈블리어
        -고급 언어(인간어에 가까운 언어): <ex>C언어, 파이썬, 자바
-Assembly language(어셈블리어): 
--복잡함(단점), CPU바뀔 시 다시 배워야함(단점)
--속도가 굉장히 빠름(장점), 바로 하드웨어에 간섭 가능(장점)

-Compiler(컴파일러)
--인간이 보기 좋다.(장점), 현대적인 프로그램의 기틀(장점), 유연성이 높고 효율 역시 높다(장점)
--어셈블리어보다는 덜 하지만 그래도 어렵다.(단점)

### 고급언어들의 특징
-인터프리터?, 컴파일러?
--인터프리터란? 
---별도의 컴파일 없이 한줄 한줄 따로 기계어로 번역함
---딱히 실행 파일이 없어서 쉽고,접근성이 높다
---근데 느리다
<예시> python
x는 90이상 100이하이다.
= [90<= x <=100]

--컴파일러란?
---실행 전에 소스코드를 모두 기계어로 번역한다.
---번역된 기계어들을 모아 exe란 파일로 만들어 저장한다.
---실행 속도가 매우 빠르다.
<예시> C
x는 90이상 100이하이다.
= 90<= x && x<=100
### 프로그램의 설계
-C언어의 실행 방식
--1. #include stdio.h
--2. 전처리기로 이동(코드를 미리 정리하는 역할)
--3. 확장 소스 프로그램으로 이동
--4. 컴파일러로 이동(인간의 언어 ---> 기계어)
--5. object program으로 이동(따로 나누어서 정리하여 오류만 뽑아냄==속도 증가)
--6. Linker로 이동(lib이랑 연결하여 실행 준비)
--7. .exe파일로 전환되어 실행
